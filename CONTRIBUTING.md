Thank you for your interest in contributing to MuOnline-Editor!

Guidelines
- Report issues using the repository's Issues page. Provide steps to reproduce, expected behavior, and screenshots when helpful.
- For code changes, open a pull request that targets the main branch.
- Keep PRs small and focused. Describe the change clearly in the PR description.

Running and testing
- This project is primarily static. To test changes locally, serve the repository directory with a simple HTTP server (see README).
- If you modify server.js, include instructions and any dependencies in a package.json.

Code style
- Use plain JavaScript and HTML consistent with existing files.
- Add comments where behavior is not obvious.

License and CLA
- By contributing you agree that your contributions will be licensed under the project's license (MIT by default).
